# ARC Studio audit — 2026-05-14

## Executive summary

The repo is **functionally healthy but CI-degraded**. The CLI, daemon, and browser shell all work correctly. All test suites pass (356 Python, 108 TypeScript, 695 SwarmGraph). Security posture is consistent with the audit doc. However, five CI workflows exist and at least three would fail if triggered today due to known, fixable issues (broken lockfile, tracked artifacts, broken package build). The most impactful single action is deleting `.env` from git tracking, which unblocks `check-pr.sh` and makes CI gates honest.

| Severity | Count |
|----------|-------|
| P0 (broken) | 3 |
| P1 (degraded) | 6 |
| P2 (papercut) | 5 |
| P3 (style) | 0 |

## Findings by severity

### P0 (broken / unsafe)

- **[F-0] `pnpm install --frozen-lockfile` fails** — `pnpm-lock.yaml` has empty `specifiers: {}` for at least one workspace package (`packages/arc-extension/`). A developer or CI runner following the README's Quick Start instructions (`pnpm install --frozen-lockfile`) hits this on first clone. The `node.yml` CI workflow works around this with `--frozen-lockfile=false`, silently masking the issue. **Fix**: Regenerate lockfile (`pnpm install --no-frozen-lockfile`) and commit the corrected `pnpm-lock.yaml`.

- **[F-1] Stale test file blocks test collection** — `python/tests/test_routes_execute.py` imports `fastapi.testclient`, which is not a project dependency. The file tests the old FastAPI server that was migrated to aiohttp. Any `pytest` run that discovers this file (e.g., `uv run pytest` without `--ignore`) will fail at collection. Currently masked because the test suite command uses `--ignore`. **Fix**: Delete `python/tests/test_routes_execute.py` (159 lines, fully superseded by `tests/web/`).

- **[F-2] `arc-arena` TypeScript build is broken** — `theia-extensions/arc-arena/` has 5 TypeScript errors (TS2305, TS2353, TS17016) related to Theia API mismatch. This blocks `pnpm -r build` (used in `bootstrap-dev.sh` and CI `node.yml`). The package is still listed in `applications/electron/package.json` dependencies, so the Electron app build is blocked too. **Fix**: Either fix the TypeScript errors, remove `arc-arena` from the workspace and electron deps, or add it to a build exclude list.

### P1 (degraded / risky)

- **[F-3] `.env` file tracked in git** (`file:.env`) — Contains `G4F_API_KEY=g4f_u_mohmn5_...`. The U-1 security decision accepts this residual risk (key rotated, repo private), but the tracked `.env` breaks `scripts/check-pr.sh` and `scripts/check-artifacts.sh`, which treat any tracked `.env` as a CI gate failure. All downstream CI gates that run these scripts will falsely fail. **Fix**: `git rm --cached .env` and add `.env` to `.gitignore`. Coordinated with U-1 acceptance.

- **[F-4] `bootstrap-dev.sh` build step fails** (`scripts/bootstrap-dev.sh:37`) — The `pnpm run build` step triggers `pnpm -r build`, which fails at `arc-arena` (F-2). A new developer following the README Quick Start verbatim hits this. The browser app `src-gen/` directory isn't created, so `pnpm start:browser:stub` also fails until manually built. **Fix**: Depends on F-2 fix.

- **[F-5] README claims "no git tags yet"** (`README.md:9`) — This is false. Tag `v0.1.0-alpha` exists. **Fix**: Update `README.md` to remove or rephrase the "no git tags" statement.

- **[F-6] `eval` command undocumented in README CLI listing** (`README.md:140-152`) — The CLI has 12 commands including `eval`, but the README's command table lists only 11. **Fix**: Add `eval` to the command list.

- **[F-7] CI `node.yml` runs PR hygiene check before installing dependencies** (`file:.github/workflows/node.yml:21-25`) — The `check-pr.sh` and bootstrap test steps run before `pnpm install --frozen-lockfile=false`. The `check-pr.sh` check for tracked artifacts is independent of npm deps so it won't error, but the bootstrap node scripts (`node tests/unit/arc-protocol.test.js`) must be pure Node.js with no external deps. This is fragile. **Fix**: Reorder CI steps: install first, then hygiene check.

- **[F-8] CI `python.yml` runs `bash scripts/check-artifacts.sh`** — This script will fail due to tracked `.env` (F-3). Any PR that triggers this workflow will fail even if code is correct. **Fix**: Depends on F-3 fix.

### P2 (papercut / drift)

- **[F-9] `arc-arena` still in `applications/electron/package.json` dependencies** — Was removed from browser deps (pre-tag hygiene) but still referenced in electron deps. Since electron app isn't the primary target, this is a papercut until someone tries to build the electron app. **Fix**: Remove `arc-arena` from electron deps too, or fix the build.

- **[F-10] `check-pr.sh` flags `packages/arc-browser-app/src-gen/` as tracked artifacts** — These are Theia-generated HTML files that need to be committed for the browser app build to work. The check script doesn't exclude them. This means `check-pr.sh` is not usable as a CI gate for any PR that touches the browser app. **Fix**: Add `src-gen/` to the exclusion list in `check-artifacts.sh` or accept that these files are required.

- **[F-11] `.tool-versions` pins `pnpm 9.12.0` but environment runs `9.15.9`** — Minor version drift. `pnpm 9.x` is backward-compatible, so this doesn't break anything currently, but a CI environment pinned to `9.12.0` might behave differently. **Fix**: Sync `.tool-versions` to `9.15.9` or whatever the CI uses.

- **[F-12] Python `mypy` strict mode shows 455 errors across 68 source files** (`pyproject.toml` sets `strict = false`, so CI doesn't catch these) — The type annotation debt is real: untyped `dict` literals, `Any` return types, `_Environ`/`dict` mismatches. Not a runtime problem but refactoring hazard. **Fix**: Gradual annotation remediation; start with the 45 auto-fixable ruff errors as a warm-up.

- **[F-13] `console.log` in production TypeScript code** (`packages/arc-extension/src/node/services/trace-parser.ts:51`) — A `console.log` call in the trace parser service. This leaks to stderr/stdout in production. **Fix**: Replace with proper logger call or remove.

1. **`git rm --cached .env` + update `.gitignore`** — Unblocks `check-pr.sh`, `check-artifacts.sh`, and all 3 CI workflows. This is the single highest-leverage fix. (Addresses F-3, F-8, F-10 partially)

4. **Fix or exclude `arc-arena`** — Either fix the 5 TypeScript errors, remove it from `applications/electron/package.json` dependencies, or add it to a build exclusion list in `package.json` scripts. Unblocks `pnpm -r build`, `bootstrap-dev.sh`, and the electron app. (Addresses F-2, F-4, F-9)

5. **Update `README.md`** — Fix "no git tags" claim, add `eval` to CLI command table, sync `pnpm` version. (Addresses F-5, F-6, F-11)

6. **Update `check-artifacts.sh` exclusions** — Add `src-gen/` and `.env` (if tracked.env is intentional per U-1) to the exclusion regex. (Addresses F-10)
