> **Scope:** IDE (Theia) + TUI (Textual) — enterprise developer-tool standards  
> **Source:** Synthesized from 7 prior research sessions (SwarmGraph, Memory/Context, Workspace Intelligence, MCP, Provider/Budget, CLI, TUI, Theia IDE audits) + CSS analysis  
> **Methodology:** Code reading, CSS analysis, contract test reading, e2e smoke analysis

## 1. UX Audit

### Information Architecture problems

**The 12-tab flat taxonomy is too wide.** ArcStudioWidget presents all 12 tabs at the same hierarchy level with no grouping: Chat · Runs · Workflows · Assurance · SwarmGraph Insight · Battle · Command Centre · MCP Workbench · Test Bench · Edit Plans · CI Guardrails · Config. A developer opening the IDE for the first time faces 12 unlabeled options with no progressive disclosure. Compare: VS Code uses a 5-icon activity bar grouping Explorer, Search, SCM, Debug, Extensions. IntelliJ uses a 6-item tool-window strip.

**Mode/profile/isolation are buried.** The most important operational context — "what mode is this running in?" — is in the bottom status strip as 11px monospaced text and in the Config tab. There is no persistent top-of-screen indicator showing `[fake] stub · local · trust:untrusted`.

**Command Centre is a dashboard, not a command interface.** It aggregates 12 arcService calls into a grid of read-only panels but has no actionable affordances — all panels say "Sessions (read-only)", Tasks says "(stub/empty)". The name implies action; the content provides status only.

**Advanced features are not progressively disclosed.** SwarmGraph Insight, MCP Workbench, and CI Guardrails are first-class tabs alongside Chat, making a power-user debugging surface appear mandatory for all users.

### TUI-specific problems

**Streaming is visually broken.** The MarkdownBlock is mounted once and never refreshed during streaming. Users see a blank or partial assistant message until the next entry is appended. This is the most visible UX failure in the TUI.

**Status bar shows wrong context limit (64k vs 200k).** `_DEFAULT_CONTEXT_LIMIT = 64_000` in `context_meter.py` but the auto-compact threshold uses `ARC_CONTEXT_LIMIT` env default of 200,000. The meter shows inflated percentages.

**HelpScreen is stale.** 12 bindings and commands (Ctrl+X, Shift+Tab, /plan, /build, /auto, /wallet, /budget, /statusline, /title, /models, /model-info, /providers) are implemented but not documented in F1.

**Ctrl+O is a no-op** (body is `pass`) but is listed in BINDINGS. Ctrl+R history search silently drops its message with no handler.

**SessionsView advertises "Enter switch" but the action is not implemented.** Users who discover the sessions panel and try to switch sessions will find the Enter key does nothing.

### IDE-specific problems

**Three critical keybinding conflicts** override core IDE defaults:
- Ctrl+E → overrides "Go to File" (the most-used VS Code keybinding)
- Ctrl+Shift+S → overrides "Save All"
- Ctrl+H → overrides "Find & Replace"

**Two identically-labeled "Open ARC Studio" commands** appear in the command palette. Users cannot distinguish the legacy arc.open from arc-studio:open.

**ArcContextDrawer exists but is unreachable** — no command, no area, no contribution class. R-AUDIT16 is marked Baseline Complete but the widget shows stub data and has no access path.

**TestBenchTab shows detected tests but has no Run button.** Users must drop to the CLI to actually run tests. The detect-only UX creates a false affordance — why show commands if you can't run them?

**ChatTab transcript is ephemeral.** It is a local React state array, lost on page reload, not persisted to the session store, and not synced with the Python session. Users expect IDE chat to be persistent.

### Cross-surface problems

**Token budget meter inconsistency.** TUI shows cost in status bar; IDE shows no cost/token meter at all. Provider/model changes are visible in TUI status bar but not in IDE status bar. Budget/wallet is TUI-only.

**AGENTS.md governance charter is invisible to running agents.** Discovered (path+SHA), capable of generating CapabilityCards, but content is never parsed or injected. The governance document governing agent behavior cannot be read by agents automatically.

**No @file/@folder mention syntax anywhere.** Every context injection surface (TUI, IDE, CLI) lacks inline mention parsing.

## 2. Enterprise IA Proposal

### Proposed activity bar hierarchy (IDE)

```
◉ ARC Studio              ← primary view (collapses into activity bar icon)
  ├── [Chat]              ← primary workflow
  ├── [Runs]              ← output browser
  └── [Workflows]         ← workflow catalog

⚙ Operations              ← secondary group, gated/collapsed by default
  ├── [Assurance]         ← HITL + audit + replay
  ├── [Edit Plans]        ← code review/apply
  └── [Command Centre]    ← aggregate status dashboard

🔬 Advanced               ← power-user group, hidden by default
  ├── [SwarmGraph Insight]
  ├── [MCP Workbench]
  ├── [CI Guardrails]
  └── [Test Bench]

⚔ Arena                   ← opt-in experimental
  └── [Battle]

⚙ Config                  ← gear icon, standard IDE pattern
```

### Information hierarchy by user type

| Persona | First-seen view | Progressive disclosure |
|---|---|---|
| New user | Chat + Workflows | Config only on first run; Runs appears after first run |
| Daily user | Chat + Runs | Assurance on HITL notification; Edit Plans on agent suggestion |
| DevOps user | Assurance + CI Guardrails | Full advanced group |
| Platform engineer | MCP Workbench + SwarmGraph Insight | All advanced group |

### Status rail (persistent top/bottom bar)

Visible at all times, replacing the buried status strip:

```
[◉ ARC]  [fake · stub]  [untrusted ⚠]  [gpt-4.1-mini · OpenAI]  ──────  [● daemon]  [$0.0042]  [F1 Help]
```

Fields: product mark · execution mode · trust level · model/provider · spacer · daemon status · session cost · help

This collapses to icons at narrow widths. The mode badge changes color by risk: fake=gray, local-safe=blue, local-paid=amber, gateway=red.

### TUI navigation proposal

```
[Plan] [Build] [Auto] ← mode cycling (Shift+Tab, already implemented)

Primary views via / or Ctrl+P:
  - / → chat (default, no navigation)
  - /runs → runs list with search
  - /hitl → HITL inbox
  - /sessions → session management with working switch

Hidden by default (accessible via command palette only):
  - /providers → provider setup
  - /settings → isolation + theme
  - /swarmgraph → (CLI-only, no TUI)
```

### Semantic color system

| Semantic token | Maps to | Usage |
|---|---|---|
| `--arc-color-live` | success | SSE connected, streaming, active |
| `--arc-color-degraded` | warning | Daemon unreachable, fallback data |
| `--arc-color-paid` | warning | Any paid-call surface |
| `--arc-color-denied` | error | Sandbox denial, policy block |
| `--arc-color-fake` | muted | Fake/offline mode indicators |
| `--arc-color-measured` | primary | Real provider call, measured cost |

### Card anatomy (standardized)

Every information card should follow this structure:

```
┌─ Card ──────────────────────────────────────────────────────────┐
│  Header row                                                      │
│  ├── Title (13px semibold)              [Status badge] [More ⋯] │
│  └── Subtitle (11px muted)                                       │
├──────────────────────────────────────────────────────────────────┤
│  Body content (13px)                                             │
├──────────────────────────────────────────────────────────────────┤
│  Footer row (11px)                                               │
│  [CLI equivalent]              [Evidence link] [Timestamp]       │
└──────────────────────────────────────────────────────────────────┘
```

CSS class conventions (already in use, formalize):
- `.arc-card` — base card (1px border, 4px radius, 12px padding)
- `.arc-card--clickable` — adds hover state, cursor:pointer
- `.arc-card--highlighted` — focus ring
- `.arc-badge` — pill badge (10px, uppercase, semibold)
- `.arc-badge--live` / `--degraded` / `--paid` / `--denied` / `--fake`
- `.arc-state-banner` — full-width banner (info/warning/error/success variants)
- `.arc-monospace` — `font-family: var(--arc-font-family-mono); font-size: 11px`

### TUI component standardization

The TUI lacks a shared `TCSS` library — each widget styles itself independently. Recommended shared tokens in `base.tcss`:

```tcss
/* Add to base.tcss: */
$paid: $warning;                    /* paid-call surfaces */
$live: $success;                    /* SSE connected */
$fake: $muted;                      /* fake/offline mode */
$card-border: $border;
$card-bg: $surface;
```

### State system (4 states, consistent across IDE + TUI)

| State | Icon | Color | Copy pattern |
|---|---|---|---|
| Empty | ○ | muted | "{Entity} will appear here after {trigger}." |
| Loading | ↻ | muted | "Loading {entity}…" |
| Error | ✗ | error | "{What failed}. {Why}. [{Action}]" |
| Degraded | ⚠ | warning | "{Entity} from {source}. Live data unavailable." |

## 5. Copy Improvements

### Principles
1. **Honest about synthetic/stub data.** Never claim real data when it's fake/offline.
2. **Name the CLI equivalent.** Every card action should note `arc <subcommand>` equivalent.
3. **State implications.** Paid calls, mutations, and sandbox denials need explicit framing before the user commits.
4. **Progressive specificity.** Short label → one-line tooltip → expandable detail.

## 6. Accessibility Gap Report

### WCAG 2.1 AA failures (confirmed)

| Violation | Component | Severity | Fix |
|---|---|---|---|
| `critical` and `high` risk map to same CSS class `info` (blue) — color-only distinction | McpWorkbenchTab `RISK_VARIANT` | **AA fail** (1.4.1 Use of Color) | Map `critical` → `danger` CSS class; add text label differentiation |
| Keyboard navigation test suite uses `expect(true).toBe(true)` — never actually tests anything | `accessibility.test.tsx` | **Test failure** | Rewrite with real component renders + axe + keyboard events |
| `SandboxInspect` input has no `<label>` or `aria-label` | CommandCentreTab | **AA fail** (1.3.1 Info and Relationships) | Add `aria-label="Command to inspect"` |
| Refresh buttons across all tabs have no `aria-label` | McpWorkbenchTab, TestBenchTab, CiGuardrailsTab | **AA fail** (4.1.2 Name, Role, Value) | Add `aria-label="Refresh MCP workbench status"` etc. |
| Risk badge spans have no `aria-label`; render "critical" as visual text only | McpWorkbenchTab | **A fail** (4.1.2) | `aria-label="Risk level: critical"` |
| `tab` button `aria-controls` pointing to panel but no visible focus ring on tab switch | ArcStudioWidget | Incomplete | Add `:focus-visible` ring on `.arc-studio-tab--active` after keyboard navigation |
| ToolCard rerun hint "r rerun" is display text in a collapsed state — not keyboard-discoverable without expanding | TUI ToolCard | Medium | Add hint to help screen; ensure 'r' key is discoverable |
| CommandPalette Ctrl+P keybinding conflicts with browser-standard Save As (in some contexts) | ArcStudioWidget | Low | Already uses `focusedView` context; acceptable |

### Confirmed accessible (✅)

- `role="tablist"` / `role="tab"` / `aria-selected` / `role="tabpanel"` structure in ArcStudioWidget ✅

- Error states use `role="alert"` in McpWorkbenchTab ✅
- Focus ring via `*:focus-visible { outline: 2px solid ... }` in arc-studio-widget.css ✅

### Slice D: Fix TUI streaming (highest-impact TUI fix)

File: python/src/agent_runtime_cockpit/tui/widgets/transcript.py

In the 100ms polling loop, after mounting new entries, also refresh the
last MarkdownBlock if streaming is active:

```python
def _poll_entries(self) -> None:
    # ... existing new-entry mounting code ...
    
    # Refresh last assistant entry during streaming
    if self.data.is_streaming and self.data.entries:
        last = self.data.entries[-1]
        if last.role == "assistant":
            try:
                blocks = list(self.query(MarkdownBlock))
                if blocks:
                    blocks[-1].update(last.content)
            except Exception:
                pass  # best-effort, never crash the poll loop
```

Where `MarkdownBlock.update(text)` calls:
```python
def update(self, content: str) -> None:
    self.renderable = Markdown(content, code_theme="monokai") if content else ""
    self.refresh()
```

    from agent_runtime_cockpit.cli_repl.slash_commands import _build_registry, get_registry
    _build_registry()
    cmds = sorted(get_registry().list_commands(), key=lambda c: (c.category, c.name))
    
    binding_lines = [
        "KEYBOARD SHORTCUTS",
        "  Enter / Shift+Enter  Send / New line",
        "  Ctrl+C (×2)          Exit",
        "  Ctrl+P / Ctrl+K      Command palette",
        "  Ctrl+L               Scroll to bottom",
        "  Ctrl+X               Activity tray (MCP/capability decisions)",
        "  Ctrl+T               Export transcript",
        "  Shift+Tab            Cycle mode: Plan → Build → Auto → Review",
        "  F1 / ?               This help",
        "  Up / Down            Input history",
        "  Tab                  Complete slash command",
        "  !<command>           Run via sandbox (local-safe policy)",
        "",
        "SLASH COMMANDS",
    ]
    for cmd in cmds:
        binding_lines.append(f"  /{cmd.name:<22} {cmd.help_text}")
    
    self.query_one("#help-text", Label).update("\n".join(binding_lines))
