## Key Findings Summary

**Five biggest problems:**

1. **MCP resources bypass the entire risk gate and audit trail** â€” `arc://runs/`, `arc://traces/`, `arc://audit/` resource reads are invisible to the D-02 policy system.

2. **`arc mcp serve` corrupts stdio framing** â€” Rich markup printed to stdout before `mcp.run(transport="stdio")` breaks every MCP client connecting to ARC.

3. **Proxy `env=None` leaks all secrets** â€” default proxy invocation with no `env` argument passes full process environment to upstream subprocess.

4. **Registry `approved_tools`/`blocked_tools` are dead code** â€” per-tool override mechanism exists in data model but never consulted by `decide_call()`.

5. **Critical and high risk are visually identical** â€” `RISK_VARIANT` maps both to `'info'`.
