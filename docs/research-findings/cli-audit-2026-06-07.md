## 2. CLI UX Critique

### Dry-run polarity is split (3 patterns)

| Command | Dry-run mechanism |
|---|---|
| `arc run --dry-run` | Explicit flag → suppresses execution |
| `arc runs prune` | Dry-run by default → `--yes` to actually execute (inverted polarity) |
| `arc batch` | Separate `batch plan` subcommand (not a flag) |

### `arc serve` has no `--json`

The only command in `exec.py` without JSON output capability. Inconsistent with all peers.

### 10 of 17 `arc runs` subcommands undocumented in README

Documented: `search`, `replay`, `fork`, `budget`, `links`  
Undocumented: `get`, `diff`, `trace`, `status`, `delete`, `export`, `import`, `backfill`, `contract`, `autopsy`

### `--approval-token` vs `--confirm <token>`

The codebase uses `--approval-token` (scoped, pre-issued, bound to command hash) — not a `--confirm` string. Users expecting `--confirm "ACTION:..."` won't find it; only `providers action` uses an inline `--confirm` token.

### `arc studio chat` live gate is softer than `arc providers action`

`arc providers action` requires 4 explicit conditions including a typed confirm token.  
`arc studio chat` live path is activated by session state set via `/connect` slash command — no CLI-level confirmation token required at invocation time.

### Undocumented flags

- `arc run --prompt`, `--runtime-mode`, `--profile-id`, `--allow-paid-calls` — not in README
- `arc runs links --filter`, `--stable-id` — not documented
- `arc providers shell --live` — functional live path, not mentioned in README
- `arc run` silently remaps runtime-mode aliases (`fake` → `fake/offline`, etc.)

## 4. Safety Review

### Strong gates ✅

| Gate | Mechanism |
|---|---|
| `arc run` live runtimes | Dual-gate: profile `allow_paid_calls` + env `ARC_<RUNTIME>_ALLOW_COSTS=true` |
| LangGraph+SwarmGraph real path | Two env vars: `ARC_REAL_RUNTIME_SMOKE=1` + `ARC_LANGGRAPH_SWARMGRAPH_REAL=1` |
| `arc sandbox run` destructive | Policy deny-by-default, exit 3, audit logged |
| MCP server | Workspace trust gate (exit 3 if untrusted) |
| Provider paid calls | `enforce_paid_call_gate` + `require_dual_gate`; typed denial events |
| `EnforcementContext.dry_run=True` | Cannot be bypassed; denies ALL gates |

### Weak or missing gates ⚠️

| Command | Gap | Severity |
|---|---|---|
| `arc policy rule-add/remove` | No confirmation, no dry-run, direct security rule mutation | **High** |
| `arc sandbox audit-compact` | No confirmation gate — prunes audit chain records | **High** |
| Run IDs (all `arc runs *`) | Not sanitized before path construction (`base_dir / f"{run_id}.jsonl"`) — path traversal possible | **High** |
| `arc audit --chain <path>` | Accepts raw `Path` with no confinement check | **Medium** |
| `arc studio chat` live mode | Activated by session state (`/connect`), no CLI confirmation token | **Medium** |
| `arc obs export --format otlp-http` | Exfiltrates trace data, gated by `--confirm-network-export` flag only (no env var) | **Low** |
| `arc providers quota reset` | No dry-run, no confirm gate | Low |
| `arc runs fork/replay` | No confirmation gates | Low |
| Credential access (`allow_if_no_db=True`) | Lenient trust bypass when no trust DB initialized | Low |

### Path traversal (HIGH — needs fix)

```python
# jsonl.py — no validation
def _run_path(self, run_id: str) -> Path:
    return self.base_dir / f"{run_id}.jsonl"
```

A `run_id` of `../../etc/passwd` would resolve outside the trace directory. No character allowlist, no `Path.resolve()` + `relative_to()` confinement check anywhere in the `runs` command surface.

## 5. JSON Schema Gap Review

### Standard envelope (compliant commands)

```json
{
  "version": "1.0",
  "ok": true,
  "data": { "..." },
  "error": null,
  "meta": {
    "duration_ms": 12.3,
    "adapter": null,
    "workspace": "/path",
    "timestamp": "2026-06-07T09:00:00+00:00"
  }
}
```

### Non-compliant files

| File | Issue |
|---|---|
| `a2a.py` | Bare `json.dumps({"ok": True, ...})` — no `version`, no `meta`, no `data`/`error` split; errors are plain strings not `{code, message}` |
| `flight.py` | `typer.echo(json.dumps(data, indent=2))` — raw dict, no envelope |
| `events.py` | `rprint(JSON(...))` or `print(json.dumps(...))` — no envelope |
| `battle.py` | Interleaves prose `typer.echo("Starting battle...")` with JSON `_out()` on stdout |
| `providers.py` | Unconditional `typer.echo("ARC Studio Provider Setup")` prose on stdout before JSON |

### Double-ok anti-pattern

`arc health --json` returns `"ok": true` at envelope level with `data.ok: false` when daemon is down. Callers must check both levels.

### Other schema issues

- `duration_ms` is `Optional[float]` — `null` in `arc version` and `arc doctor swarmgraph` snapshots
- `version: "1.0"` in `DYNAMIC_KEYS` normalization list — envelope version regressions won't be caught by snapshots
- `workspace` not populated in `arc version`, `arc health`, `arc bug-report` metadata
- `_out()` emits clean JSON to non-tty stdout even without `--json` flag — `--json` only affects formatting, not gate

### Snapshot coverage

Only 4/85+ commands have golden snapshots: `version`, `health`, `status_empty`, `doctor_swarmgraph`. `status_empty` snapshot is CI-skipped (flaky — runtime detection ordering).

# CLI Next Slice: Run ID Safety + JSON Compliance + CI Discoverability

## Context

ARC Studio v0.8-r-ux2. Four CLI gaps discovered:

1. Run IDs are used directly in path construction without sanitization.
   `base_dir / f"{run_id}.jsonl"` — path traversal is possible.

2. a2a.py, flight.py, and events.py emit ad-hoc JSON bypassing ArcEnvelope.
   These commands are not usable by scripts expecting the standard envelope.

3. audit-cli-help.py tests 75 commands but is NOT in CI. Help regressions
   are only caught manually. test_cli_smoke.py's _commands() discovery
   misses all 50 add_typer() sub-apps entirely.

4. arc policy rule-add and arc policy rule-remove mutate security rules
   with no confirmation gate, dry-run, or approval-token.

## Scope

### 1. Run ID sanitization (path traversal fix)

File: python/src/agent_runtime_cockpit/storage/jsonl.py
      (also audit chain paths in audit.py)

Add a validation function:
```python
_SAFE_RUN_ID_RE = re.compile(r'^[A-Za-z0-9][A-Za-z0-9_\-\.]{0,127}$')

def _validate_run_id(run_id: str) -> str:
    if not _SAFE_RUN_ID_RE.match(run_id):
        raise ValueError(f"Invalid run_id: {run_id!r}")
    return run_id
```

Call it at the top of every method that constructs a path from run_id:
`_run_path`, `load`, `save`, `trace_path`, `load_contract`, `load_autopsy`.

Also add `candidate.resolve().relative_to(self.base_dir.resolve())` check
after path construction as defense-in-depth.

For `audit verify/export/query --chain <path>`:
Validate that the resolved path is within `~/.arc/audit/` or the workspace
`.arc/audit/` directory. Reject with `INVALID_INPUT` error and exit 1.

Tests: add to `test_cli_runs.py`:
- `test_run_id_path_traversal_rejected` — `arc runs get "../etc/passwd"` exits 1
- `test_run_id_with_special_chars_rejected` — `arc runs get "run;rm -rf"` exits 1
- `test_run_id_valid_characters_accepted` — `arc runs get "run-abc123"` exits 0

### 2. Fix ArcEnvelope compliance for a2a.py, flight.py, events.py

def _commands(app):
    """Recursively collect all commands including add_typer() sub-apps."""
    found = []
    for name, cmd in app.registered_commands:
        found.append([name])
    for name, sub_app in app.registered_groups:
