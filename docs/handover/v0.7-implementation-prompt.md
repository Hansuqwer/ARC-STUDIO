## Scope (3 independent opt-in features, each defaults OFF)

### Feature 1 — Opt-in pricing feed (Task 1 in spec)

**NOT Ed25519. NOT `api.arc-studio.ai`. Uses hash-pinning.**

```python
# New file: python/src/agent_runtime_cockpit/cloud/pricing_feed.py

@dataclass(frozen=True)
class PricingFeedConfig:
    feed_url: str = "https://openrouter.ai/api/v1/models"   # primary
    feed_source: str = "openrouter"                          # "openrouter" | "models-dev"
    refresh_interval_hours: int = 168                        # weekly
    pinned_hash: str | None = None   # SHA-256 of last accepted feed
    cache_path: Path | None = None

    @property
    def enabled(self) -> bool:
        return os.environ.get("ARC_PRICING_FEED_ENABLED") == "1"
```

**Failure modes (all silent, fail-closed):**
- Network down → keep local `VENDOR_CONFIGS` snapshot, log `[catalog] offline, using local`
- Parse failure → keep local
- Hash mismatch → keep pinned + warn user once via REPL
- Server 5xx → keep local

**Hash-pinning flow:**
1. Fetch feed (only if `ARC_PRICING_FEED_ENABLED=1`)
2. SHA-256 the raw response
3. If hash != `pinned_hash`: emit one-time warning, prompt `arc pricing-feed accept-new-hash`
4. If accepted: update `~/.arc/pricing_feed_cache.json` + update pinned hash
5. Parse using the **same** `parse_openrouter()` / `parse_models_dev()` from
   `scripts/sync_from_pricing_feed.py` (single source of truth — reuse dev-time parser)

**12 required tests** (`tests/cloud/test_pricing_feed.py`):
1. feed disabled by default → no network call
2. network timeout → local snapshot returned
3. hash mismatch → discarded, warn emitted, local kept
4. hash match → cache updated
5. parse failure → local kept
6. stale cache used when offline
7. `ARC_PRICING_FEED_SOURCE=models-dev` → different parser path
8. `--source openrouter` default path
9. cache written to correct path
10. SHA-256 computed deterministically
11. first-run with no cache → local snapshot
12. NEW: `test_source_switch_openrouter_to_models_dev_falls_back`
