# 07 — Run Contract

## Summary

The RunContract is a pre-run agreement that bounds what a run can do: cost ceiling, allowed tools, write scope, rollback plan. It is proposed before execution, accepted by the user, and checked on completion.

```python
from pydantic import BaseModel, Field
from typing import Literal, Optional

class RunContract(BaseModel):
    contract_id: str
    run_id: Optional[str] = None        # Assigned after run starts
    session_id: str
    objective: str                       # Human-readable goal
    runtime: str
    mode: Literal['plan', 'build', 'auto']
    allowed_tools: list[str] = Field(default_factory=list)
    write_scope: list[str] = Field(default_factory=list)  # Paths/globs
    cost_ceiling_usd: float | Literal['unknown'] = 'unknown'
    approval_policy: str = 'default'     # Policy name or path
    rollback_plan: Literal['git-revert', 'manual', 'none'] = 'git-revert'
    evidence_expected: list[str] = Field(default_factory=list)
    status: Literal['proposed', 'accepted', 'fulfilled', 'violated'] = 'proposed'
```

## Lifecycle

```
PROPOSED ──► ACCEPTED ──► FULFILLED
                 │                ▲
                 ▼                │
              VIOLATED ──────────┘
```

1. **PROPOSED** — Generated by `JobSupervisor.start_run()` before execution begins
2. **ACCEPTED** — User accepts via API (`POST /api/runs/contract/{id}/accept`)
3. **FULFILLED** — Run completes within contract terms
4. **VIOLATED** — Run exceeds cost, writes outside scope, or uses unapproved tools
