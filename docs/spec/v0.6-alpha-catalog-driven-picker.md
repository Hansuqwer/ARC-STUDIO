# v0.6-alpha — Catalog-Driven Model Picker + Capability UI

> **Release type:** feature
> **Scope:** UI surface that consumes the OpenRouter (or models.dev fallback)
> catalog to populate a `/models` slash command + capability-aware widgets.
> Connection metadata stays hand-coded per policy
> (`docs/research/catalog-driven-connection-tradeoffs.md`).
> **Source:** Catalog data synced via `scripts/sync_from_pricing_feed.py`
> (introduced in v0.5.1-alpha-v3).

---
