# v0.7-alpha — Opt-In Cloud Features (Pricing Feed, Shared Broker, Observability Bridge)

> **Release type:** feature (additive, opt-in)
> **Scope:** Three independent opt-in features that integrate with hosted
> services while preserving local-first default behavior. Each defaults OFF.
> Each requires explicit user configuration. Each shows a visible indicator
> when active. None changes any existing user's behavior unless they
> actively enable it.
> **Source / origin:** `docs/policy/local-first.md` §"Opt-in escape hatches"
> identified three queued features. This spec implements them.
> **Tag rationale:** v0.7 (not v0.6) so v0.6 can stay focused on OTel
> deprecation + token-estimator accuracy work that's already queued.

#### §A. Design

```python
# cloud/pricing_feed.py
from __future__ import annotations

import json
import logging
from dataclasses import dataclass, field
from datetime import datetime, timedelta, timezone
from pathlib import Path
from typing import Mapping

import httpx  # opt-in dep; only loaded when feature configured

from ..providers.cost_rate import CostRate
from ..events import PricingFeedRefreshed, emit

log = logging.getLogger(__name__)


@dataclass(frozen=True)
class PricingFeedConfig:
    feed_url: str | None = None          # ARC_PRICING_FEED_URL
    signature_pubkey_path: Path | None = None  # ARC_PRICING_FEED_PUBKEY
    refresh_interval_hours: int = 168     # default: weekly
    cache_path: Path | None = None        # local cached feed for offline use

    @property
    def enabled(self) -> bool:
        return self.feed_url is not None and self.signature_pubkey_path is not None


@dataclass
class FeedRefreshResult:
    success: bool
    fetched_ts: datetime
    feed_version: str | None = None
    rows_updated: int = 0
    rows_added: int = 0
    error_reason: str | None = None      # set if signature failure / network failure


class PricingFeedClient:
    """Signed-JSON pricing feed. Off by default. Fails closed: signature
    failure → keep last-known-good local table; never silently use unsigned data."""

    def __init__(self, config: PricingFeedConfig) -> None:
        self._config = config

    def refresh_if_due(self, current_rows: Mapping[str, CostRate]) -> FeedRefreshResult:
        if not self._config.enabled:
            return FeedRefreshResult(success=False, fetched_ts=datetime.now(timezone.utc),
                                     error_reason="feature disabled")
        if not self._is_due():
            return FeedRefreshResult(success=False, fetched_ts=datetime.now(timezone.utc),
                                     error_reason="not yet due")
        try:
            response = httpx.get(self._config.feed_url, timeout=10)
            response.raise_for_status()
            payload = response.json()
            if not self._verify_signature(payload):
                log.warning("pricing feed signature invalid; keeping local table")
                return FeedRefreshResult(success=False, ...,
                                          error_reason="signature_invalid")
            updated, added = self._apply(payload, current_rows)
            emit(PricingFeedRefreshed(
                feed_url=self._config.feed_url,
                feed_version=payload.get("version"),
                rows_updated=updated,
                rows_added=added,
            ))
            return FeedRefreshResult(success=True, ..., feed_version=payload.get("version"),
                                      rows_updated=updated, rows_added=added)
        except Exception as exc:
            log.warning(f"pricing feed refresh failed: {exc}")
            return FeedRefreshResult(success=False, ...,
                                      error_reason=f"network_failure: {type(exc).__name__}")
```

#### §B. Feed signature scheme

- Feed format: JSON `{"version": "2026-Q3", "fetched_ts": "...", "rows": [...], "signature": "ed25519:..."}`
- ARC verifies signature against pubkey at `ARC_PRICING_FEED_PUBKEY` path
- Failure modes: signature invalid → keep local; network failure → keep local; parse error → keep local
- Each failure logged + emits no event (we don't tell observers about non-events)

```python
# cloud/budget_broker.py
@dataclass(frozen=True)
class BrokerConfig:
    broker_url: str | None = None         # ARC_BUDGET_BROKER_URL
    auth_token: str | None = None         # ARC_BUDGET_BROKER_TOKEN
    team_id: str | None = None            # ARC_BUDGET_TEAM_ID
    fallback_to_local: bool = True        # if broker unreachable, use local cap

    @property
    def enabled(self) -> bool:
        return all([self.broker_url, self.auth_token, self.team_id])


class BudgetBrokerClient:
    """Shared budget for team mode. Hooks into BudgetEnforcer.preflight()
    to query remote cap. Falls back to local on any failure."""

    def remote_preflight(self, scope: BudgetScope, amount: Decimal) -> BrokerCheckResult:
        if not self._config.enabled:
            return BrokerCheckResult(approved=True, fell_back=True,
                                      reason="broker disabled; using local")
        try:
            response = httpx.post(f"{self._config.broker_url}/preflight",
                                  json={"team_id": self._config.team_id,
                                        "scope": scope.name, "amount": str(amount)},
                                  headers={"Authorization": f"Bearer {self._config.auth_token}"},
                                  timeout=5)
            response.raise_for_status()
            result = response.json()
            return BrokerCheckResult(
                approved=result["approved"],
                remote_remaining=Decimal(result.get("remaining", "0")),
                fell_back=False,
            )
        except Exception as exc:
            if self._config.fallback_to_local:
                log.warning(f"broker unreachable, fallback to local: {exc}")
                return BrokerCheckResult(approved=True, fell_back=True,
                                          reason=f"broker_unreachable: {type(exc).__name__}")
            else:
                return BrokerCheckResult(approved=False, fell_back=False,
                                          reason=f"broker_unreachable_no_fallback")
```

#### §B. Integration with `BudgetEnforcer.preflight()`

```python
# budget/schema.py — additive change
def preflight(self, scope: BudgetScope, amount: Decimal) -> PreflightResult:
    local = self._local_preflight(scope, amount)
    if not local.approved:
        return local  # local cap already says no; don't bother broker
    if self._broker_client.enabled:
        remote = self._broker_client.remote_preflight(scope, amount)
        if not remote.approved:
            return PreflightResult(approved=False, reason="team_cap_exceeded",
                                    broker_remaining=remote.remote_remaining)
        emit(BudgetBrokerSync(
            scope=scope.name, amount_usd=amount,
            local_approved=True, remote_approved=remote.approved,
            fell_back=remote.fell_back,
        ))
    return local
```

### Task 3 — Observability bridge (~4h)

#### §A. Design

```python
# cloud/observability_bridge.py
@dataclass(frozen=True)
class BridgeConfig:
    destination_url: str | None = None    # ARC_OBSERVABILITY_BRIDGE_URL
    protocol: str = "otlp"                # otlp | langfuse | helicone | custom
    auth_header: str | None = None        # for hosted services
    consent_mode: str = "per_session"     # per_session | always | env_only

    @property
    def enabled(self) -> bool:
        return self.destination_url is not None


class ObservabilityBridge:
    """Exports local OTel traces to user-configured destination.
    Per-session consent required unless ARC_OBSERVABILITY_BRIDGE_CONSENT=yes."""

    def __init__(self, config: BridgeConfig, session_consent: bool = False) -> None:
        self._config = config
        self._session_consent = session_consent

    def export_span(self, span: ReadableSpan) -> bool:
        if not self._config.enabled:
            return False
        if not self._has_consent():
            return False
        try:
            # Use otel-exporter-otlp if installed (optional extra)
            from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
            exporter = OTLPSpanExporter(
                endpoint=self._config.destination_url,
                headers={"Authorization": self._config.auth_header} if self._config.auth_header else None,
            )
            exporter.export([span])
            emit(ObservabilityExportStarted(
                destination=self._config.destination_url,
                protocol=self._config.protocol,
                span_count=1,
            ))
            return True
        except ImportError:
            log.warning("observability bridge configured but otel-exporter-otlp not installed; "
                        "pip install agent-runtime-cockpit[observability]")
            return False
```

#### §B. Consent gating

Three modes:
- `per_session` (default when feature enabled): interactive prompt on first export per session
- `always`: opt-in once via config; no prompts
- `env_only`: `ARC_OBSERVABILITY_BRIDGE_CONSENT=yes` env var only, ignored if absent

|---|---|
| Malicious feed claims Claude is $0.001/M; user runs $1000 of calls thinking it's $1 | Ed25519 signature verification; pubkey pinned by user in config; failure = fail-closed |
| Feed server compromised; injects valid-signed but wrong data | Out of scope (key compromise = compromise; same as compromised auth token anywhere). Mitigation: ARC project publishes feed integrity log; user can diff feeds across time |
| Feed server tracks usage via fetch patterns | Mitigation: ARC fetches once per week, no request body; only fetches the public URL. No usage data transmitted. |
| User can't reach feed (offline) | Mitigation: cached feed used; local table is always primary; feed is enrichment |

### Shared budget broker

| Threat | Mitigation |
|---|---|
| Broker compromised; approves everything | User can see /wallet shows team-mode-active; can disable opt-in |
| Network silently fails; team cap exceeded because ARC fell back to local | Configurable: `fallback_to_local=False` makes it fail-closed instead |
| Auth token leaked | Token rotation; out of scope for ARC; user manages |
| Broker tracks which instance spent what (privacy) | Documented: yes, broker sees per-instance spend; user opts in knowing this |

### Observability bridge

| Threat | Mitigation |
|---|---|
| User accidentally exports PII / secrets in spans | Already redacted via `redactDict` at trace-write time (v0.4.x); but document that user is responsible for evaluating destination's data handling |
| Destination compromised; spans leaked | User chose destination; ARC is integration layer only |
| Per-session consent prompt fatigue causes user to always-accept | Three consent modes (per_session / always / env_only) give graduated friction |

Full version with 1-page-per-feature in `docs/threat-models/v0.7-opt-in.md`.
