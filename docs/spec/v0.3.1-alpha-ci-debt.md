## 4. Task-by-task

### Task 1 — Gap audit (~5 min)

```bash
cd packages/arc-protocol-ts

# Confirm thresholds currently low
grep -nE "branches|functions|lines|statements" jest.config.js | head -10
# expect: 35 / 42 / 44 / 43

# Confirm run-events.ts has no test file yet
ls -la src/run-events.test.ts 2>&1 || echo "MISSING (expected)"

# Confirm run-events.ts is the file that grew
wc -l src/run-events.ts src/run-diff.ts src/swarmgraph-ir.ts

# Confirm KnownRunEvent + KNOWN_RUN_EVENT_TYPES are exported
grep -n "KnownRunEvent\|KNOWN_RUN_EVENT_TYPES\|parseRunEvent\|isKnownEvent" src/run-events.ts
```

**STOP if** any of the above disagrees with expectations. Report mismatch before touching code.

### Task 2 — Backfill TS tests (~3-4 hrs)

Create `packages/arc-protocol-ts/src/run-events.test.ts` covering, at minimum:

| Suite | Cases |
|---|---|
| `parseRunEvent` happy path | one case per variant in `KnownRunEvent` union (use `protocol/fixtures/run-event-registry.json` as source of truth) |
| `parseRunEvent` unknown type | returns `null` or unknown-typed default per current impl |
| `parseRunEvent` extra fields | `extra="ignore"` parity — extra payload keys don't throw |
| `parseRunEvent` malformed payload | missing required field returns `null`/throws-consistently |
| `isKnownEvent` true cases | every `KNOWN_RUN_EVENT_TYPES` member |
| `isKnownEvent` false cases | unknown string, empty string, null |
| `KNOWN_RUN_EVENT_TYPES` shape | set equals exported list; no dupes |
| SwarmGraphCostData cache fields round-trip | with + without `cache_read_input_tokens` / `cache_creation_input_tokens` (R-03 forward-compat — already covered by Python side, add TS parity) |

**Target coverage on `run-events.ts` after this:** ≥90% lines, ≥85% branches.

### Task 3 — Raise thresholds (~10 min)

After Task 2 lands and `pnpm --filter arc-protocol-ts test --coverage` reports new actuals, edit `packages/arc-protocol-ts/jest.config.js:11-14`. Rules:

- Set each threshold to `floor(actual) - 2` (2pt headroom; do NOT chase the actual exactly or one removed test breaks CI).
- If actual is still <70 on any axis, **STOP and report** — don't paper over more debt with low thresholds. Instead, write more tests in Task 2 first.

Example target:

```js
coverageThreshold: {
  global: {
    branches: 72,   // pinned 2pt below restored actual ~74%
    functions: 88,  // pinned 2pt below restored actual ~90%
    lines: 76,      // pinned 2pt below restored actual ~78%
    statements: 73, // pinned 2pt below restored actual ~75%
  },
},
```

(Adjust to your actuals.)
