## 4. Decision log additions

**Why OpenRouter beats models.dev for this sprint specifically:**
- 50%+ more model coverage in our target vendors (verified by OpenRouter's
  300+ models claim vs models.dev's ~100; we'll know exactly after Task 0.5)
- 4 of 6 ARC schema extension fields are NATIVE in OpenRouter (is_free,
  deprecation_date, canonical_slug, tiered pricing) — less custom logic
- No auth; edge-cached; production-grade

**Why we don't switch ARC at runtime to OpenRouter as a router:**
- That's v0.5.0+ territory (we already have openai_compatible.py with
  Gemini block; OpenRouter could be added as another vendor block later)
- This sprint is purely about pricing data acquisition, not request routing

**Why models.dev stays as fallback:**
- OpenRouter is a commercial company; one source of failure
- models.dev is community + opencode-backed; different failure mode
- `--source models-dev` is one CLI flag; trivial to maintain
