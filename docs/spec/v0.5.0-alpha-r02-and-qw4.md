# v0.5.0-alpha — R-02 Compaction Triage + QW-4 MCP Handle Virtualization

> **Feature release.** First *behavior-changing* sprint in the token-saving
> series. Combines two cohesive features:
>
> 1. **R-02 — Deterministic context compaction triage.** When context usage
>    approaches `context_limit × threshold` (default 0.85), evict middle
>    message-pairs (Lost-in-the-Middle informed) until back under 0.70.
>    System prompt + current user message + first/last N pairs always
>    preserved. **No LLM in decision path (CoSAI).**
>
> 2. **QW-4 — MCP output handle virtualization.** When a tool returns >N
>    bytes (default 8KB), store full content as `arc://output/sha256/<hex>`
>    handle and surface a `resource_link` to the model with head+tail
>    preview. `/expand <handle>` re-injects on demand. Backed by SQLite
>    blob store under the v0.4.1 budget DB.
>
> Per `TOKEN_SAVING_PLAN-2.md` §15 and the research deliverables
> `docs/research/R-02-compaction-options.md` + `docs/research/QW-4-mcp-handle-design.md`.

@command(name="expand", help="/expand <handle-prefix> [--filter <regex>]")
def cmd_expand(args: list[str], ctx: SlashContext) -> str:
    if not args:
        return "Usage: /expand <handle-prefix> [--filter <regex>]"
    prefix = args[0]
    filter_regex = _parse_filter(args[1:])
    try:
        content = ctx.handle_store.expand(prefix)
    except HandleNotFound:
        return f"[handle {prefix} not found; rerun the originating tool call]"
    except HandleCorrupt:
        return f"[handle {prefix} corrupt; rerun the originating tool call]"
    if filter_regex:
        content = _apply_filter(content, filter_regex)
    # Inject as next user message; R-02 may compact first if needed
    ctx.inject_user_message(content.decode(errors="replace"))
    return f"[expanded {len(content)} bytes from {prefix}]"

### Task 4 — R-02 Compaction Strategy (~4h, ~320 LOC)

Per `docs/research/R-02-compaction-options.md` §3:

```python
# context/compaction.py
from dataclasses import dataclass
from typing import Sequence

from ..protocol.message import Message  # whatever the canonical message type is


@dataclass(frozen=True)
class CompactionConfig:
    trigger_threshold: float = 0.85   # compact when used/limit >= this
    stop_threshold: float = 0.70      # stop when used/limit < this
    keep_first_n_pairs: int = 2       # preserve first N user/assistant pairs
    keep_last_n_pairs: int = 4        # preserve last N user/assistant pairs
    # System prompt + current user message ALWAYS preserved.


@dataclass(frozen=True)
class CompactionResult:
    messages_kept: Sequence[Message]
    messages_evicted: Sequence[Message]
    tokens_before: int
    tokens_after: int


def compact(
    messages: Sequence[Message],
    context_limit: int,
    context_used: int,
    config: CompactionConfig = CompactionConfig(),
    token_counter: TokenCounter = ...,
) -> CompactionResult:
    """Deterministic Lost-in-the-Middle eviction. NO LLM IN PATH."""
    if context_used / context_limit < config.trigger_threshold:
        return CompactionResult(messages, [], context_used, context_used)
    # Identify pairs
    pairs = _identify_pairs(messages)
    # Always-keep set
    keep_pairs = set(pairs[:config.keep_first_n_pairs]) | set(pairs[-config.keep_last_n_pairs:])
    # Evict middle pairs in order until under stop_threshold
    evicted = []
    for pair in pairs:
        if pair in keep_pairs:
            continue
        if _compute_used(messages, evicted, token_counter) / context_limit < config.stop_threshold:
            break
        evicted.append(pair)
    kept = [m for m in messages if m not in _flatten(evicted)]
    return CompactionResult(
        messages_kept=kept,
        messages_evicted=_flatten(evicted),
        tokens_before=context_used,
        tokens_after=_compute_used(kept, [], token_counter),
    )
```

```
1. Start a session.

   event fires.
4. Verify next request to provider has fewer messages (middle pairs evicted).
5. Verify response quality still reasonable (sanity check — not a benchmark).
```

### Smoke 3 — Combined: compaction evicts TO handle
```
1. Start a session with attached large tool output (virtualized via QW-4).
2. Continue session until compaction triggers.
3. Verify: evicted messages with handle references are NOT actually deleted
   from disk (handle store is separate from message history).
4. Verify: `/expand` still works on those handles after compaction.
```

### Smoke 4 — Token savings measurement
```
For 3 known workloads (pick representative samples):
  - Long-context coding agent (many file reads)
  - Multi-turn research conversation
  - Tool-heavy data analysis loop

Record pre/post:
  - Total input tokens per turn
  - Total output tokens per turn
  - Cache hit rate (gen_ai.usage.cache_read.input_tokens / total)
  - Cost USD per session
  - User-observable quality (anecdotal: did the response degrade?)

Expected wins per research deliverables:
  - QW-4 on tool-heavy: 40-90% input token reduction on virtualized turns
  - R-02 on long sessions: 30-60% input token reduction once compaction triggers
  - Combined: cache hit rate should also improve (smaller, more stable prefixes)
```
