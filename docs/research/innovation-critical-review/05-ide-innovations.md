## Diagnosis

The Theia plan is sane: Chat, Graph, Runs, Review/Apply, Config, status bar, graph in main area, chat/right sidebar, no default Trace UI. It is not yet a cockpit. The IDE must make relationships visible: graph node -> chat turn -> tool output -> approval -> diff -> evidence -> receipt.

## IDE Concepts

| Concept | UI Surface | v0.1 Reservation | Why It Matters |
|---|---|---|---|
| Graph/Chat Cross-Highlighting | Graph, Chat, Runs, Review | Stable `event_id`, `node_id`, `message_id` links. | Creates a coherent cockpit feel quickly. |
| Evidence Cards | Chat and Runs | `evidence_refs[]` on messages and run summaries. | Makes claims verifiable. |
| Visual Run Contract | Chat pre-run card and Runs detail | `run_contract` field. | Unifies HITL, cost, trust, runtime, audit. |
| Fulfillment Report | Runs detail and receipt | `contract_result` field. | Shows promises kept/violated. |
| Failure Autopsy Panel | Runs detail, Chat failure card | `failure_autopsy` object. | Replaces raw Trace UI for most users. |
| Trust Diff Sheet | Config / Workspace Trust / Provider changes | Policy diff taxonomy. | Makes safety changes reviewable. |
| Handoff Workbench | Phase boundary v0.2 | `handoff_schema_ref`. | Future multi-runtime killer feature. |
| Runtime Capability Inspector | Config runtime tab | Manifest capability fields. | Runtime selection becomes guarantees, not names. |
| Review/Apply Cockpit | Review panel + Theia diff | Link diff hunks to run contract and evidence. | Review is not isolated from agent intent. |
| Cockpit Command Palette | Theia command palette provider | Action descriptors by session state. | Lets the IDE surface the next useful action. |

## Theia Layout Changes

- Keep the Theia-native shell. Do not build a custom three-column IDE.
- Add a **Run Contract strip** above the chat input only when a run is staged or active.
- Add **Evidence tab/section** inside RunSummary, not a separate global panel.
- Make **NodeInspector** the bridge: node metadata, current state, last events, evidence, allowed commands.
- Add **AutopsyCard** as the first expanded area on failed run rows.
- Add **TrustDiffModal** before trust/policy/provider changes.

## Graph/Chat/Evidence Interaction Model

1. User clicks a graph node.
2. Chat scrolls to the relevant agent message and tool cards.
3. Runs panel filters event summary to that node.
4. Review panel highlights diffs generated by that node.
5. Evidence cards show files/tests/audit/tool output linked to that node.

This is the lowest-cost way to make ARC feel unlike Cursor/Windsurf without shipping replay.

## IDE Anti-Patterns To Reject

| Anti-pattern | Reason |
|---|---|
| More default tabs | The current plan already risks panel sprawl. |
| Raw trace event timeline as default | Violates v0.1 decision and overwhelms users. |
| Cursor-style inline diffs as primary differentiation | Competitors own this surface; ARC should be evidence/contract-led. |
| Hidden policy changes in settings | Trust changes must be reviewed like code changes. |
| Graph-only wow demo | A graph that cannot explain or command is decorative. |

## v0.1 Lock Criteria Additions

- Graph nodes and chat/tool cards share stable IDs.
- RunSummary supports `receipt_path`, `contract`, `autopsy`, `evidence_refs`.
- Config changes that weaken policy must generate Trust Diff data, even if UI starts basic.
- Review hunks carry `origin_node_id` and `origin_decision_id` when available.
