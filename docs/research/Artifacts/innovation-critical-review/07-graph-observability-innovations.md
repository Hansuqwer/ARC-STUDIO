## Core Critique

A live graph is not automatically a cockpit. If users cannot ask the graph what happened, command nodes, compare paths, or link graph events to evidence, the graph is a screensaver.

## Node Inspector Requirements

Every node inspector should show:

- Node ID, label, type, runtime, state.
- Current contract scope.
- Inputs/outputs as redacted summaries.
- Evidence cards generated by node.
- Tool calls and approvals linked to node.
- Cost and trust boundary crossings.
- Allowed commands now vs reserved commands.
- Failure autopsy if failed.

## Run Comparison

Run comparison should not be raw trace diff. Normalize to:

- Graph topology changes.
- Prompt/intent changes.
- Runtime/model/provider changes.
- Policy/trust decisions.
- Cost/call count changes.
- Failure node and recovery path.
- Evidence coverage changes.

## Graph UX Details

- Use color for state, but never color alone.
- Active edge animation should mean state transition, not decoration.
- Node badges should include `evidence`, `risk`, `cost`, `blocked`, `contract violated`.
- Keep minimap for large graphs, but add semantic filters: failed, waiting, wrote files, crossed trust boundary, produced evidence.
- Avoid exposing event JSON in node inspector by default; link advanced trace.

## Before Replay Exists

ARC can still ship:

1. Stable node IDs.
2. Current run state.
3. Last event summary.
4. Node-linked chat/evidence/diff.
5. Explain edge via transition event summary.
6. Autopsy on failed node.
7. Contract status overlay.

That is enough to make graph useful in v0.1.

## Rejected Graph Ideas

| Idea | Why |
|---|---|
| Particle animations | Adds spectacle, not control. |
| Full replay scrubber in v0.1 | Conflicts with scope and needs storage semantics. |
| Runtime-specific graph hacks | Breaks cross-runtime cockpit. |
| Graph as separate app | Coherence comes from linking graph/chat/runs/review. |
