# v0.5.1-alpha OpenRouter Adoption Postmortem

> addendum prediction of ~40.

## Predicted vs actual

| Metric | Addendum prediction | OpenRouter actual | Delta |
|---|---|---|---|
| Total rows across 6 vendors | ~40 | 91 | +128% |
| Free-tier rows | ~3 (vendor-direct) | 4 (OpenRouter `:free` variants — different semantics) | see note |
| Deprecated rows flagged | 5–10 | 1 (`deepseek-r1`) | −80% |
| Cross-check warnings | <20 | 91 | all "new since 2026-Q2 snapshot," benign |

## Key findings

1. **OpenRouter coverage is materially broader than the addendum.** 91 rows vs 40 predicted. Qwen alone had 49 models — the addendum only captured 7.

2. **OpenRouter does NOT consistently set `deprecation_date` for retired legacy IDs.** Only `deepseek-r1` had it. `deepseek-chat` (retires 2026-07-24) and `kimi-k2-0905` (expired 2026-05-25) had no `deprecation_date` field. Addendum overlay (`8ce6f39`) was still required for correctness. This is a systematic gap — OpenRouter's deprecation signaling is unreliable.

3. **OpenRouter `:free` variants ≠ vendor-direct free tiers.** `glm-4.5-air:free`, `kimi-k2.6:free`, `qwen3-next:free`, `qwen3-coder:free` are OpenRouter gateway rate-limited variants. The actual GLM-4.7-Flash from Zhipu direct (no-cost, no rate limit) is a separate offering. `/wallet` now distinguishes via `(via OpenRouter free tier; rate-limited)`.

4. **Moonshot aggressively prunes legacy IDs.** The addendum predicted 4–5 legacy Kimi K2 SKUs; OpenRouter had only 1 (`kimi-k2-0905`). Either Moonshot delisted quickly or OpenRouter pruned stale rows. Either way, the addendum's legacy model list was stale within ~30 days of publication.

## Implications for next quarterly snapshot

- Refresh `pricing-snapshot-2026Q3.md` from OpenRouter as primary source.
- Manually re-verify `deprecation_date` against vendor docs — OpenRouter is unreliable here. Use addendum overlay as the correction layer.
- Track which vendors prune legacy IDs aggressively (Moonshot) vs retain them (DeepSeek) — affects overlay maintenance burden.
- Consider adding a `source_verified_at` field to `CostRates` for freshness tracking in a future sprint.
