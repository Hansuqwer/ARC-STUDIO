# ARC Mobile Runtime — Real vs Mock Matrix

ARC Mobile Runtime is currently a **simulator preview**. The implemented runtime is safe, local, static/mock-oriented infrastructure for planning, policy, simulation, and trace inspection. It is **not** a production native mobile SDK yet.

## Current truth

- No production native iOS or Android bridge exists.
- No real camera, microphone, contacts, calendar, photos, files, health, or location access exists.
- No autonomous background execution exists.
- No production MCP gateway exists.
- Built-in capabilities are mock/simulator capabilities.
- CLI simulation predicts and records what a plan would do; it does not execute native device APIs.

## Matrix

| Area | Current status | Classification | Production implication |
|---|---|---|---|
| Python mobile models | Implemented Pydantic models for manifests, capabilities, action plans, reports, traces, and policy surfaces | Real simulator foundation | Useful for validation and tooling; not native execution |
| Capability catalog | Built-in `.mock` capabilities only | Real mock catalog | Safe for demos and tests; not real device access |
| Manifest loading | Loads `arc-mobile-capabilities.json` into typed models | Real simulator tooling | Needs strict/enterprise validation before production use |
| Validation | Fail-closed MVP rules for sensitive native access, network, background, MCP exposure, secrets, and hashes | Real safety foundation | Must be extended before native access |
| Simulator | Static/mock plan analysis and prediction | Real simulator | Does not execute device APIs |
| Policy explain | Allow/deny explanation over capabilities and plans | Real simulator policy | Missing enterprise/user/device context |
| Trace recorder | JSONL simulated event recorder with hashes | Real MVP trace | Not a signed production audit log yet |
| Runtime-pack bridge | Generates ARC runtime-pack metadata from mobile manifest | Real bridge | Target-specific pack generation still needed |
| TypeScript protocol | Type/interface mirror and light guards | Real type surface | Needs generated runtime validators |
| Expo package | Mock-mode TypeScript stub | Stub/preview | Not a production Expo Module |
| React Native package | Mock-mode TypeScript stub | Stub/preview | No TurboModule or JSI bridge |
| Flutter package | Package shell/stub | Stub/preview | No Dart platform interface or native plugin implementation |
| iOS native bridge | Not implemented | Missing | Required before iOS production SDK |
| Android native bridge | Not implemented | Missing | Required before Android production SDK |
| App-store privacy artifacts | Not generated by production tooling | Roadmap | Required before real permissions |
| Enterprise control plane | Not implemented | Missing | Required before enterprise label |

## Naming rule

Until real mobile framework bridges, signed plans, scoped approvals, compliance artifacts, and production traces exist, refer to this as:

**ARC Mobile Runtime Simulator Preview**

Avoid calling it production-ready, enterprise-ready, or a real native mobile SDK.
