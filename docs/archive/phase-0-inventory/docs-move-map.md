# Phase 0 — Docs Inventory and Move Map

Status: DRAFT (Phase 0 inventory, non-destructive)
Scope: every `*.md` file under `docs/` and at repo root.
Output: the exact move map Phase 1 will execute. No files are touched in Phase 0.

## How to fill this file

1. `git ls-files '*.md' 'docs/**/*.md'` and copy paths.
2. For each, classify: Keep-canonical | Rename | Archive | Pointer-only | Delete-after-Phase-1.
3. Target path is the Phase 1 destination, or `—` for Keep-canonical / Delete.
4. Pointer-only files become ≤10-line MOVED stubs (format below).

## Classification legend

| Disposition | Meaning |
|---|---|
| Keep-canonical | Stays at current path, content updated normally |
| Rename | Moves to a new path; old path gets Pointer-only stub |
| Archive | Moves to `docs/archive/...`; old path gets Pointer-only stub if externally referenced |
| Pointer-only | Old path becomes ≤10-line MOVED stub |
| Delete-after-Phase-1 | Removed after Phase 1 acceptance; no pointer kept |

## Pointer file format (locked)

```text
# MOVED

This document has moved to: docs/<new-path>.md
Reason: docs consolidation (Phase 1).
Date: <YYYY-MM-DD>
Old content archived at: docs/archive/<path>
Do not edit this file. Edit the canonical location.
```
