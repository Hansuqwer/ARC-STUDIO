# ARC Studio v0.1.0-alpha Release Notes

**Release Date:** 2026-06-01  
**Status:** Alpha Release  
**Target Users:** Early adopters, developers, AI researchers

---

## Overview

ARC Studio v0.1.0-alpha is the first public release of the Agent Runtime Cockpit — an IDE and runtime environment for developing, executing, and monitoring AI agent workflows. This alpha release provides a solid foundation for local agent development with SwarmGraph, comprehensive CLI tooling, and a browser-based IDE built on Eclipse Theia.

**What's Ready:**
- ✅ Full-featured CLI for agent workflow execution and management
- ✅ Browser-based IDE with live event streaming and visualization
- ✅ SwarmGraph native runtime with fake/offline mode (no API keys required)
- ✅ Provider-backed runtime foundations with Anthropic Claude support
- ✅ Multi-turn conversational sessions with tool use
- ✅ Comprehensive cost tracking and budget enforcement
- ✅ Session management and persistence
- ✅ Workspace trust and security controls

**What's Deferred:**
- ❌ Broad provider-backed adoption (limited to narrow gated paths)
- ❌ LM Arena live product features (stub/default only)
- ❌ Production-ready multi-user deployment
- ❌ Electron desktop packaging (foundations exist, no release artifact)

## Key Features

### 1. CLI-First Development Experience

**Unified Command Interface:**
```bash
arc --help                    # Comprehensive CLI help
arc inspect                   # Detect agent runtimes in workspace
arc runtimes --capabilities   # List available runtimes with honest capability reporting
arc run <workflow>            # Execute workflows with full observability
arc studio chat               # Interactive REPL with session persistence
arc eval                      # Evaluate runs against golden traces
```

**Session Management:**
- Persistent chat sessions with automatic save/resume
- Legacy session migration support
- Bare `arc` command launches interactive REPL in TTY
- Session-level runtime mode and tool configuration

### 2. Provider-Backed Runtime with Tool Use

**Multi-Turn Conversations:**
- Streaming responses with incremental text deltas
- Sequential tool execution with iteration limits
- Trust-tagged tool results (ADR-019)
- Cost tracking with per-call component breakdown

**Built-in Tools:**
- `read_file` — Read file contents (untrusted output)
- `list_directory` — List directory entries (untrusted output)
- `get_current_time` — Get current timestamp (trusted output)

**Tool Management:**
```bash
/tools list              # List available tools
/tools enable <tool>     # Enable specific tool
/tools disable <tool>    # Disable specific tool
```

### 3. SwarmGraph Native Runtime

**Fake/Offline Mode:**
- No API keys required for development
- Deterministic behavior for testing
- Full event emission and observability
- Native Python runtime (no subprocess overhead)

**Adoption Protocol:**
- Narrow local-real path with explicit gates
- Honest capability reporting (no false claims)
- Clear separation between fake/offline and provider-backed modes

### 4. Cost Tracking and Budget Enforcement

**Comprehensive Cost Accounting:**
- Per-call cost breakdown with component tracking
- Measured vs. estimated cost sources
- Token-level usage tracking (input, output, cache creation, cache read)
- Decimal-based arithmetic (ROUND_HALF_EVEN, 8-decimal precision)

**Budget Controls:**
- Session-scoped and global budget caps
- First-launch confirmation gates
- Paid-call opt-in requirements
- Real-time cost accumulation

### 5. Browser-Based IDE

**Live Event Streaming:**
- Real-time event stream from running workflows
- Live/replay/disconnected state handling
- Configured daemon URL support with auto-discovery
- SSE-based transport with terminal event semantics

**Visualization:**
- SwarmGraph topology and consensus visualization
- Run timeline with event filtering
- Cost panel with provider/model/token breakdown
- Assurance tab for HITL and audit material

### 6. Security and Trust

**Workspace Trust:**
- External trust database (`~/.arc/trusted-workspaces.json`)
- Explicit trust confirmation for new workspaces
- Trust metadata on legacy session content

**Injection Pattern Scanner:**
- Heuristic detection of prompt injection attempts
- Code execution, data exfiltration, role-play patterns
- Structured content scanning for tool results
- `--bypass-injection-scan` flag for advanced users

**Isolation Providers:**
- None (direct subprocess, no filtering)
- Subprocess (env-filtered, blocked secrets)
- Docker provider foundations (not production-ready)

### 7. Runtime Modes

**Three Canonical Modes:**
1. **fake** — Deterministic offline mode, no API calls
2. **gated_local** — Local execution with explicit gates
3. **provider_backed** — Live provider calls with budget enforcement

**Runtime Capability Introspection:**
```bash
arc runtimes --capabilities --json
```
Returns honest capability report with:
- Detected runtimes and availability
- Required environment variables
- Paid-call requirements
- Fake/offline support status
- Provider-backed status

## Installation

### Prerequisites

- **Node.js:** >= 18.0.0
- **pnpm:** >= 8.0.0
- **Python:** >= 3.11
- **uv:** Latest (for Python package management)

### Quick Start

```bash
# Clone repository
git clone https://github.com/Hansuqwer/arc-theia-studio.git
cd arc-theia-studio

# Install dependencies
pnpm install --frozen-lockfile

# Build all packages
pnpm build

# Install Python CLI
cd python
uv pip install -e .

# Verify installation
arc --version
arc --help

# Start browser IDE (optional)
cd ../applications/browser
pnpm start
```

### Python CLI Only

```bash
cd python
uv pip install -e .
arc --help
```

### Browser IDE Only

```bash
pnpm install --frozen-lockfile
pnpm build
cd applications/browser
pnpm start
```

## What's Not Included

### Scope Boundaries (Non-Negotiable)

**No Broad Provider-Backed Adoption:**
- SwarmGraph adoption is fake/offline by default
- Narrow local-real path exists with explicit gates
- No broad live/provider-backed execution claims

**No Adapter-Wide Keyed Audit:**
- Audit chain material exists for specific paths
- Not every run path writes/verifies keyed audit material
- Assurance tab shows honest present/missing/degraded states

**No Production/Multi-User Deployment:**
- Single-user local development focus
- No tenant isolation or concurrent user support
- Daemon auth is optional bearer-token middleware

**No LM Arena Product Feature:**
- Stub/default arena behavior only
- Gated live path exists but not product-ready
- No live arena claims in release docs

**No Electron Release Artifact:**
- Electron packaging foundations exist (ADR-008)
- PyInstaller daemon spike complete (20MB binary)
- No signed/notarized release artifact for v0.1

- Expanded provider support beyond Anthropic
- Write tools and shell/subprocess/network tools

- Parallel tool execution
- Skills system
- Web search and vision/computer-use APIs

## Known Issues

### CI/Build

- **Node/E2E:** Webpack `Aborted (core dump)` on Ubuntu CI (workaround: `NODE_OPTIONS=--max-old-space-size=8192`)
- **Python 3.12:** 52 web/daemon tests return HTTP 500 on Ubuntu (pass on Python 3.11/macOS)

### Tests

- **Provider Smoke Test:** `test_providers_action_all_gates_pass_closed_smoke` gated behind `ARC_RUN_PAID_SMOKE=1` (requires valid API key)
- **Real Runtime Smoke:** LangGraph+SwarmGraph local-real tests require both `ARC_REAL_RUNTIME_SMOKE=1` and `ARC_LANGGRAPH_SWARMGRAPH_REAL=1`

### IDE

- **Theia Async Warnings:** Expected KeybindingContribution/MenuContribution/QuickAccessContribution warnings (documented, harmless)
- **Widget Activation:** "Widget was activated, but did not accept focus after 2000ms" warning (known, non-blocking)

### Runtime

- **Provider-Backed Mode:** Requires external CLI subprocess (native runtime is fake/offline only)
- **Tool Output Limits:** Default 64KB limit with truncation marker
- **Mixed Trust Tools:** Type contract exists, runtime wrapper raises `NotImplementedError` (deferred to Phase 7+)

## Migration Guide

### From Pre-Alpha Versions

**Session Migration:**
```bash
# Migrate legacy flat sessions to canonical format
arc studio sessions migrate

# Or use compatibility alias (deprecated)
arc studio sessions-migrate
```

**Runtime Mode Changes:**
- `offline` → `fake`
- `local-real` → `gated_local`
- `live` → `provider_backed`

Legacy mode strings are accepted with deprecation warnings.

**Provider Configuration:**
- G4F (GPT4Free) provider entries removed (breaking change)
- Migrate to direct provider entries (OpenAI, Anthropic, etc.)
