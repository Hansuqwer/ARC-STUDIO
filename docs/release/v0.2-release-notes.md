# ARC Studio v0.2.0 — Release Notes (DRAFT)

ARC Studio v0.2.0 is the first release that crosses from "single-runtime prototype" to "multi-runtime cockpit with auditable, enforceable execution."

## Highlights

- **Streaming audit verification** handling 100 MB traces in bounded memory with HMAC signing
- **Discriminated TypedRunEvent union** with 20+ variants replacing v0.1 ad-hoc event shapes
- **Five typed denial events** + `POLICY_BYPASS_WARNING`, wired into JobSupervisor, MCP server start, workspace prompt loading, tool/shell invocation, and ProviderClient boundary
- **UI confirmation dialogs** for enforcements with allow-once/allow-session options
- **Consolidated CLI** with per-command modules, golden-help-output enforcement, and contextvar propagation
- **Shared adapter test harness** and `ProviderClient` interface
- **Three new adapters**: LangChain (LCEL pipelines), Anthropic SDK, OpenAI-compatible provider (covers OpenAI, Together, Groq, DeepInfra, Fireworks, local llama.cpp)

## Breaking changes

| Change | v0.1 behavior | v0.2 behavior | Migration |
|--------|--------------|--------------|-----------|
| Paid-call gate | `ARC_<RUNTIME>_ALLOW_COSTS` env-var only | Requires `--allow-paid` CLI flag | Add `--allow-paid` to CLI invocations |
| Workspace trust for prompts | No trust gate | `enforce_workspace_trust` enforced | Pass `--trust-workspace` per workspace |
| RunEvent type | Arbitrary `event_type` fields | Typed `kind` field on 20+ variants | Use typed constructors from `protocol.events` |
| Daemon MCP server start | No trust gate | `enforce_workspace_trust` enforced | Same as workspace trust |

## Additions (non-breaking)

- `POLICY_BYPASS_WARNING` event variant (ADR-0022.1)
- `--dry-run`, `--non-interactive` flags
- `arc audit query --kind <kind>` filter
- `arc-runtime-cockpit[langchain]`, `[anthropic]`, `[openai_compatible]` extras groups
- TypeScript `isPolicyBypassWarning` type guard

## Deprecations

- **Env-var-only paid-call gate** — still works alongside `--allow-paid` in v0.2, emits deprecation warning. Removed in v0.3.
- **Haystack 1.x** — not supported. Pin to `haystack-ai>=2.29`.

## What v0.2.0 does NOT do

- Does not provide a sandbox for code execution (smolagents ships in v0.3)
- Does not exhaustively enforce paid-call gates for adapter plugins that bypass `ProviderClient` — such bypasses emit `POLICY_BYPASS_WARNING`
- Does not ship MCP client support (Phase 35 — v0.3)
- Does not include LangGraph improvements beyond v0.1
- Does not provide automated upgrade tooling from v0.1

## Migration checklist

1. Add `--allow-paid` to CLI invocations calling paid providers
2. Run `arc --trust-workspace` once per existing workspace
3. If constructing `RunEvent` directly, migrate to typed constructors
4. If subscribing to SSE event stream, tolerate new `POLICY_BYPASS_WARNING` variant
5. Headless/CI: pass `--allow-*` flags upfront
